# USDB跨链

目的：每条链的难度不同，但USDB同质，可以通过BTC作为可行中转来实现USDB资产在不同链上的转移

矿工证可以设置不饿修改的USDB链ID，不设置链ID为0


## 基本原理

跨链转账：

链0上的用户A，向链1上的用户B，转账1000个USDB
1）调用链0的智能合约（其实是一个销毁操作），并得到proof
2）链0的矿工，安规则，会定期向BTC Rollup链0的状态
3）链1上的用户B,使用proof向链1的的智能合约发起取现操作
4）该智能合约的内部，会根据智能合约的内容，基于BTC的区块信息进行验证
   如果验证通过，则取现成功，用户B的余额，增加1000 USDB

核心理念是：链1上的用户在取现的时候，能验证确实有一条链在BTC上

## 一些约束

- 上述流程是不可逆的，也就是说即使用户B不取现，用户A也拿不回这1000个USDB
- 核心算法是链x的矿工，需要定期向BTC Rollup状态。流程如下

矿工A在高度n出块，这是一个rollup高度（USDB链总是可以准确的查询到给定rollup高度的rollup信息）
随后，矿工A必须在高度 n + x 的时间内（通常有5个BTC区块），写入高度n时的链Rollup信息
下个rollup高度，出块矿工B会检查矿工A是否正确提交了rollup(只要rollup信息保存在BTC链上即可)，如果没有:
    - 则要进行1+n次rollup(rollup必须是顺序的)
    - 矿工A会损失能量X，矿工B的能量会增加X
    - 矿工A会被记录在失信矿工名单上，一段时间内无法出块

## Rollup信息

Rollup信息本质上是一个默克尔树的跟节点
Proof是一个默克尔树的路径

## 可能的攻击

> 链 100, 只在BTC链上创建虚假的rollup信息

攻击起点：任何人都可以在btc上写入rollup信息，btc无法验证该rollup的有效

任何链的取现合约的实现：
1）判断proof是否用过了
2）对proof属于rollup进行校验（一个有效的默克尔树路径）
3）随后对链x 的rollup是否有效进行校验
- 链x的所有矿工证的余额必须超过一个阈值
- 链x的所有矿工证的能量必须超过一个阈值
- Rollup信息中有包含usdb-chain-x相关的PoW信息（一个完整的区块？）
